# Phase Close Checklist Template

Copy this checklist into each phase's living-checklist.md under "Phase Close" section.

---

## Phase Close Checklist — Phase [N]: [Title]

### Build Verification
- [ ] `npx tsc --noEmit` passes (zero TypeScript errors)
- [ ] `npm run build` succeeds
- [ ] `npm test` (Playwright) — all tests passing
- [ ] No regressions in existing demos/examples/themes

### Documentation
- [ ] Session log finalized (`phase-N/session-log.md`)
  - All actions numbered and committed
  - Decisions documented with rationale
  - Issues found and resolutions noted
  - Metrics: start vs. end of phase
- [ ] Living checklist updated (`phase-N/living-checklist.md`)
  - All items marked complete or deferred
  - Deferred items specify destination phase
- [ ] Retrospective written (`phase-N/retrospective.md`)
  - Composite score /100 with dimension breakdown
  - Historical score table (all phases)
  - What shipped vs. what didn't (honest)
  - Lessons learned
  - Carry-over items listed
- [ ] Wiki page created (`wiki/NN-phase-N-title.guide.html`)
  - Matches existing wiki format
  - Links to adjacent phase pages

### Project Files
- [ ] CLAUDE.md updated
  - Phase status: CLOSED with score
  - Metrics reflect current state
  - Phase Roadmap table updated
- [ ] README.md updated (if features changed)
- [ ] Memory updated (project_phase_status.md)

### Next Phase
- [ ] Phase N+1 preflight created (`phase-N+1/README.md`)
  - Grounding report with current metrics
  - Sprint breakdown with effort estimates
  - ADRs listed with titles and sprint assignments
  - Testing plan (new tests per sprint, total target)
  - Carry-over from current phase
  - Enforcement section referencing ADR-011
  - Research/implementation/testing plan per sprint
- [ ] Phase N+1 human feedback file reviewed (if exists)

### Git
- [ ] All changes committed with descriptive messages
- [ ] Pushed to origin/main
- [ ] Vercel deployment verified (if applicable)

### Score Assignment
- [ ] Visual Quality: __/20
- [ ] Functionality: __/20
- [ ] Code Quality: __/20
- [ ] Content Richness: __/20
- [ ] Developer Experience: __/20
- [ ] **Composite: __/100**

## Quick Reference: What Goes Where

| Artifact | Location | When |
|----------|----------|------|
| Every action/commit | `phase-N/session-log.md` | Real-time during phase |
| Deliverable status | `phase-N/living-checklist.md` | Real-time during phase |
| Honest scores | `phase-N/retrospective.md` | At phase close |
| Public summary | `wiki/NN-phase-N-title.guide.html` | At phase close |
| Project status | `CLAUDE.md` | At phase close |
| Next phase plan | `phase-N+1/README.md` | At phase close |
| Phase memory | `memory/project_phase_status.md` | At phase close |
