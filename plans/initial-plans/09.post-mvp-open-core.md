# 09. Post-MVP Open Core Requirements

**Purpose:** Transform Hey Bradley from a demo into a usable tool with real LLM integration, enhanced specs, and user content
**Status:** Backlog (after capstone, Summer-Fall 2026)
**Prerequisite:** MVP presentation complete (Phase 8)

---

## 1. LLM Integration (HIGH PRIORITY)

### Chat Mode — Real AI Commands

Replace canned chat commands with Claude API integration:
- Natural language understanding for all builder actions
- "Make the hero bigger with a dark gradient background" → multiple patches applied
- "Write copy for a fitness studio" → LLM generates section content
- Context-aware: LLM knows current config state, can reference existing sections
- Streaming responses with typewriter display

### Listen Mode — Real Voice

Replace simulated input with Web Speech API:
- Continuous STT (speech-to-text) in Listen mode
- Voice → text → LLM → JSON patch → live preview
- Real-time orb response (amplitude-linked, not timer-based)
- "Hey Bradley, add a testimonials section with 5-star reviews" → works

### Copy Suggestions

- Click any text element → LLM suggests 3 alternative copy options
- Tone controls: Professional, Casual, Bold, Minimal
- Industry-aware: knows the site type and adjusts vocabulary

## 2. Enhanced Spec Output (HIGH PRIORITY)

### What the LLM Did Better (from spec-review-findings.md)

These are the highest-priority post-MVP items based on the spec→HTML roundtrip test:

| # | Enhancement | Impact |
|---|-------------|--------|
| 1 | **Serif heading font options** — Add Instrument Serif, Playfair Display, Fraunces | Closes biggest visual gap |
| 2 | **Section eyebrow labels** — Small uppercase label above each heading ("WHAT WE DO") | Agency-quality pattern |
| 3 | **Fluid typography** — `clamp()` for heading sizes instead of fixed breakpoints | Professional scaling |
| 4 | **More generous section spacing** — `py-20 md:py-24` default instead of `py-16` | Better breathing room |
| 5 | **CSS design tokens** — Convert to custom properties for shadows, radii, spacing | Systematic, exportable |

### Core Pillar Specs (HUMAN + AISP)

Generate structured specification documents from the current JSON:

**HUMAN Spec Documents:**
1. **North Star** — Vision, PMF, user personas, success criteria (from site metadata)
2. **Architecture** — Tech stack, section hierarchy, component model, data flow
3. **Implementation Plan** — Page-by-page build instructions, section order, variant specs
4. **Design Bible** — Color palette, typography, spacing, component patterns

**AISP Spec Documents:**
1. **Crystal Atom Platinum** — Full `@aisp` formatted specification (already exists)
2. **Section-level specs** — Individual spec per section type with constraints
3. **Verification report** — Automated checks (contrast ratios, responsive, completeness)

### Spec Round-Trip Validation

- Generate spec from JSON → Feed spec to Claude API → Compare generated HTML against Hey Bradley preview
- Automated fidelity scoring: how closely does the LLM reproduction match?
- Target: 90%+ fidelity on structure, 80%+ on visual similarity

## 4. Design Lock Toggle

### Concept

Toggle in TopBar: **"Lock Design"** (separate from Theme Lock)

When design is locked:
- Layout, variant, section order, and spacing are frozen
- Only content changes allowed: text, images, videos
- New sections cannot be added or removed
- Variant switching is blocked
- Chat/Listen commands that affect layout are rejected: "Design is locked — only content changes allowed"

When unlocked (default):
- Everything editable as normal

### Use Case

User has finalized their design layout and is now in "content mode" — replacing placeholder text/images with real content. Design lock prevents accidental layout changes during this phase.

Works across all modes: Builder, Chat, Listen.

### Phase: Full Image Library Build

**Goal:** 200 curated images across all design styles

**Categories (target counts):**

| People/Portraits | 20 | Diverse professionals, teams, headshots |
| Creative/Art | 15 | Design, photography, studio, gallery |
| Architecture/Real Estate | 15 | Buildings, interiors, modern spaces |
| Events/Social | 10 | Conferences, parties, stages, crowds |
| Education/Academic | 10 | Classrooms, libraries, campus, research |
| Abstract/Patterns | 15 | Gradients, textures, geometric, minimal |
| Lifestyle | 15 | Travel, fitness, fashion, everyday |
| Product/E-commerce | 20 | Product shots, flat lays, packaging |

**Metadata per image:**
- `url` — Unsplash/Pexels URL with size params
- `alt` — Descriptive alt text
- `category` — Primary category
- `tags` — 3-5 searchable tags
- `style` — Design style (minimal, bold, warm, cool, dark, light)
- `suitableFor` — Section types where this image works best (hero, gallery, team, etc.)
- `mood` — Emotional tone (professional, playful, serious, warm, dramatic)

**LLM Integration (post-MVP):**
When LLM builds a site, it uses metadata to select the most appropriate image:
- Matches `suitableFor` to the section type
- Matches `mood` to the site's overall tone
- Matches `style` to the active theme

### Video Library Expansion

- Same metadata structure as images
- Categories: abstract/motion, nature, tech, business, lifestyle
- Optimized for web: <5MB, 720p, 10-30 seconds, loop-friendly
