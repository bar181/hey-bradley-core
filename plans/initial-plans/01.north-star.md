## 1. VISION

> **"A whiteboard that listens, builds what you describe in real-time, and secretly writes enterprise specs behind the scenes."**

Hey Bradley is a JSON-driven marketing website specification platform. It produces two outputs simultaneously: a live visual preview the user interacts with, and enterprise-grade AISP specification documents the user exports. It is **not** a website builder competitor ‚Äî it is a specification platform that feeds builders, dev tools, and agentic pipelines.

## 3. SUCCESS CRITERIA

### 3.1 POC Success (Capstone Demo)

The primary success factor is **wow factor**. This is a Harvard capstone project reviewed by an expert UI designer. The bar: "This looks like a funded startup product, not a student project."

| Criterion | Target |
|-----------|--------|
| Time to first meaningful result | < 30 seconds (click vibe ‚Üí see website) |
| Mode switching (DRAFT ‚Üî EXPERT) | Instant, smooth, no data loss |
| Listen Mode visual impact | "The dark screen with the glowing red orb" |
| Spec output quality | Pasting into Claude Code produces matching site |
| Design quality | Indistinguishable from Framer/Linear/Vercel's UI |

### 3.2 The "Aha!" Moments (Ordered)

1. **Click a vibe ‚Üí website appears** (DRAFT mode, 2 seconds)
2. **Toggle to EXPERT ‚Üí full property inspector appears** (same data, more control)
3. **Toggle to LISTEN ‚Üí theatrical dark mode with red orb** (wow factor)
4. **Click DATA tab ‚Üí see the JSON that drives everything** (engineering credibility)
5. **Click XAI DOCS ‚Üí see AISP specs auto-generated** (invisible documentation)
6. **Click WORKFLOW ‚Üí see the AI pipeline in action** (XAI transparency)
