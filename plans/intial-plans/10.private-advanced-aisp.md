# 10. Private Advanced AISP — Intent-Driven Specifications

**Purpose:** Advanced AISP tier with LLM-powered intent detection, contextual reasoning, and adaptive spec generation
**Status:** Backlog (after open core post-MVP)
**Prerequisite:** Post-MVP open core complete (LLM integration, core pillar specs)
**Note:** Full AISP intent spec will be provided separately — this document outlines the scope and architecture

---

## 1. Concept: Intent-Driven AISP

### Current State (Open Core)

The open core AISP generates specs from explicit user configurations:
- User selects theme, adds sections, writes copy → AISP spec reflects those choices
- The spec describes **what was built**, not **why it was built**
- No understanding of user goals, audience, or business context

### Advanced State (Private)

Advanced AISP adds an **intent layer** that captures:
- **Why** the user is building this site (business goal, audience, conversion target)
- **Who** the target audience is (demographics, psychographics, technical sophistication)
- **What** success looks like (KPIs, conversion metrics, engagement targets)
- **How** the design decisions relate to the intent (why this layout, why these colors, why this copy tone)

### Example

**Open Core AISP:**
```
Ω := { Render marketing website: 8 sections, theme:professional, mode:light }
```

**Advanced AISP with Intent:**
```
Ω := { 
  Render marketing website: 8 sections, theme:professional, mode:light,
  intent := {
    goal: "Generate qualified B2B leads for consulting services",
    audience: "VP/C-suite at mid-market companies ($50M-$500M revenue)",
    conversion: "Schedule a consultation call via Calendly",
    tone: "Authoritative but approachable, data-driven",
    competitors: ["McKinsey", "Bain", "BCG"],
    differentiator: "Senior partners only, no junior consultants"
  }
}
```

## 2. Intent Capture

### How Intent Is Gathered

1. **Chat conversation** — LLM asks targeted questions during the build process:
   - "Who is this website for?"
   - "What's the main action you want visitors to take?"
   - "What makes you different from competitors?"
   - "What tone should the copy have?"

2. **Listen mode conversation** — Same questions, voice-driven

3. **Document upload** — Brand guidelines, business plans, pitch decks → LLM extracts intent

4. **Implicit signals** — The LLM infers intent from user actions:
   - Chose "professional" theme → likely B2B
   - Added pricing section → conversion-focused
   - Added team section → trust-building is important
   - Used corporate imagery → enterprise audience

### Intent Schema

```typescript
interface AISPIntent {
  goal: string                    // Primary business objective
  audience: {
    primary: string              // Target demographic
    secondary?: string           // Secondary audience
    sophistication: 'low' | 'medium' | 'high'  // Technical level
  }
  conversion: {
    primaryCTA: string           // Main conversion action
    secondaryCTA?: string        // Secondary action
    urgency: 'low' | 'medium' | 'high'
  }
  brand: {
    tone: string[]               // e.g., ["professional", "warm", "data-driven"]
    competitors: string[]        // Named competitors
    differentiator: string       // Key USP
  }
  constraints: {
    timeline?: string            // "Launch by Q2 2026"
    budget?: string              // "Bootstrap" | "Funded" | "Enterprise"
    compliance?: string[]        // ["GDPR", "ADA", "SOC2"]
  }
}
```

## 3. Intent-Informed Spec Generation

### How Intent Changes the Spec

**Copy generation:** Instead of placeholder text, the LLM writes copy informed by intent:
- "Schedule a Consultation" instead of "Get Started" (because intent.conversion.primaryCTA = consultation)
- "Trusted by Fortune 500 leaders" instead of "Trusted by teams" (because intent.audience = C-suite)

**Section selection:** LLM recommends sections based on intent:
- B2B lead gen → Hero + Features + Social Proof + Pricing + FAQ + CTA (conversion funnel)
- Portfolio → Hero + Gallery + About + Contact (showcase funnel)
- Blog → Hero + Featured Posts + Categories + Newsletter (engagement funnel)

**Design decisions:** LLM explains design rationale in the spec:
- "Blue palette selected: conveys trust and stability, appropriate for B2B financial services audience"
- "Testimonials use star ratings: social proof pattern optimized for conversion"

**Verification:** Intent-aware quality checks:
- V5: VERIFY CTA text matches intent.conversion.primaryCTA
- V6: VERIFY tone(copy) aligns with intent.brand.tone
- V7: VERIFY section_order optimizes for intent.conversion funnel

## 4. Private Tier Features

These features are NOT in the open core:

| Feature | Description |
|---------|-------------|
| Intent capture conversation | Guided Q&A to extract business context |
| Intent-informed copy generation | LLM writes copy based on goals/audience |
| Competitor analysis | LLM researches competitor sites and positions accordingly |
| Conversion funnel optimization | Section order optimized for intent.conversion |
| A/B variant suggestions | LLM suggests 2-3 hero/CTA variants for testing |
| Brand voice extraction | Upload brand doc → extract tone, vocabulary, restrictions |
| Design rationale in spec | Each design decision annotated with "why" |
| Intent verification rules | AISP Epsilon checks that output matches intent |
| Multi-page spec | Intent-driven page structure (Home, About, Pricing, Blog) |

## 5. Architecture

### Intent Storage

Intent is stored alongside the MasterConfig:
```typescript
interface ProjectWithIntent {
  config: MasterConfig        // Existing JSON config
  intent: AISPIntent          // New intent layer
  conversation: Message[]     // Chat/voice history that produced the intent
  specVersion: string         // AISP version with intent tier
}
```

### Spec Tiers (Updated)

| Tier | Name | Ambiguity | Intent | Open Core |
|------|------|-----------|--------|-----------|
| Bronze | Basic | <20% | No | Yes |
| Silver | Structured | <10% | No | Yes |
| Gold | Detailed | <5% | No | Yes |
| Platinum | Crystal Atom | <2% | No | Yes |
