## 1. DESIGN IDENTITY

### 1.1 Aesthetic Direction: "Warm Precision"

Hey Bradley's visual language is **warm, editorial minimalism with engineering credibility.** Think: a Scandinavian design studio crossed with a Bloomberg terminal. The warmth makes it approachable (grandma). The precision makes it credible (enterprise). The monospace typography makes it technical (Framer user).

```
WARM          →  Cream backgrounds, orange accents, soft borders, rounded cards
EDITORIAL     →  Generous whitespace, typographic hierarchy, monospace labels
PRECISION     →  Pixel-perfect alignment, consistent spacing, no decoration without purpose
ENGINEERING   →  Status bars, pipeline views, JSON syntax highlighting, version badges
```

### 1.2 What Makes This Unforgettable

The **one thing** someone will remember after a 30-second demo:

> *"The dark screen with the glowing red orb — and then suddenly a website appeared."*

Listen Mode is the theatrical centerpiece. Everything else is a well-designed tool. Listen Mode is the **moment**.

### 1.3 Anti-Patterns (What This Is NOT)

| Don't | Why |
|-------|-----|
| Purple gradients on white | Generic AI aesthetic (Jasper, Copy.ai, every Y Combinator demo) |
| Dark mode everything | Loses the warm approachability; Listen mode needs contrast |
| Rounded bubbly everything | Loses engineering credibility |
| Blue accent color | Too cold, too generic (Figma, Notion, Linear all use blue) |
| Excessive drop shadows | Looks dated; use borders and background shifts instead |
| Hero images in the builder | The builder IS the product; no stock photography |

## 3. TYPOGRAPHY

### 3.1 Font Stack

```css
/* Primary UI font — clean, modern, warm */
--font-ui: 'DM Sans', system-ui, sans-serif;

/* Monospace — for labels, status, code, JSON */
--font-mono: 'JetBrains Mono', 'Fira Code', 'SF Mono', monospace;

/* Preview/Reality content — matches user's theme font */
--font-preview: var(--user-font-family, 'Inter', sans-serif);
```

**Why DM Sans over Inter:** Inter is the most overused font in AI products. DM Sans has similar legibility with slightly more warmth and personality — rounder terminals, more humanist proportions. It pairs naturally with the warm color palette.

### 3.2 Type Scale

| Role | Font | Size | Weight | Transform | Tracking | Usage |
|------|------|------|--------|-----------|----------|-------|
| Page heading | DM Sans | 18px | 600 | none | normal | "Project Explorer", "Draft Builder" |
| Section label | Mono | 12px | 500 | UPPERCASE | 0.05em | "LAYOUT", "CONTENT", "STYLE", "STRUCTURE" |
| Accordion header | Mono | 12px | 500 | UPPERCASE | 0.05em | Section accordion titles |
| Property label | Mono | 11px | 400 | UPPERCASE | 0.03em | "DISPLAY", "DIRECTION", "PADDING" |
| Property value | Mono | 12px | 500 | none | normal | "flex", "column", "24px" — right-aligned |
| Input text | DM Sans | 14px | 400 | none | normal | Form fields, text inputs |
| Tab label | Mono | 12px | 500 | UPPERCASE | 0.08em | "REALITY", "DATA", "XAI DOCS", "WORKFLOW" |
| Status bar | Mono | 11px | 400 | UPPERCASE | 0.03em | "READY AISP SPEC V1.2" |
| Version badge | Mono | 10px | 500 | UPPERCASE | normal | "V1.0.0-RC1" — in colored badge |
| Chat placeholder | DM Sans | 14px | 400 | none | normal | "Tell Bradley what to build..." |
| JSON line numbers | Mono | 12px | 400 | none | normal | Muted color, right-aligned |
| JSON content | Mono | 13px | 400 | none | normal | Syntax highlighted |
| AI typewriter | Mono | 14px | 400 | none | normal | Orange accent color, appears letter by letter |
| Pipeline step title | DM Sans | 16px | 600 | none | normal | "Voice Capture", "Intent Parsing" |
| Pipeline step subtitle | DM Sans | 13px | 400 | none | normal | "Captured 3 intents" — muted color |

### 3.3 Typography Rules

1. **All structural labels are monospace + uppercase.** This creates the "engineering dashboard" feel that differentiates Hey Bradley from consumer website builders
2. **All user-editable content uses DM Sans.** This creates warmth in the controls
3. **All code/data uses monospace.** JSON, AISP specs, status bar, pipeline log
4. **Never use more than 2 font sizes in a single panel.** Hierarchy comes from weight and case, not size explosion

## 5. SPACING SYSTEM

### 5.1 Base Grid: 4px

All spacing uses multiples of 4px. Common values:

| Token | Value | Usage |
|-------|-------|-------|
| `space-1` | 4px | Inline icon gaps, tight element spacing |
| `space-2` | 8px | Between related elements (label → input) |
| `space-3` | 12px | Card internal padding (compact), property row height buffer |
| `space-4` | 16px | Standard card padding, section gaps |
| `space-5` | 20px | Panel padding |
| `space-6` | 24px | Between control groups |
| `space-8` | 32px | Major section breaks |
| `space-10` | 40px | Tab bar height, accordion header height |
| `space-12` | 48px | Top bar height |

## 6. MOTION & ANIMATION

### 6.1 Transition Defaults

```css
/* Standard transition for all interactive elements */
--transition-fast: 150ms ease-out;    /* Hovers, toggles, small state changes */
--transition-normal: 200ms ease-out;  /* Panel collapse, tab switch */
--transition-slow: 300ms ease-out;    /* Listen mode overlay, major state changes */
--transition-spring: 300ms cubic-bezier(0.34, 1.56, 0.64, 1);  /* Bouncy feedback */
```

### 6.2 Animation Catalog

| Animation | Duration | Easing | Usage |
|-----------|----------|--------|-------|
| Toggle pill slide | 150ms | ease-out | DRAFT↔EXPERT, LISTEN↔BUILD |
| Tab underline slide | 200ms | ease-out | Tab switching |
| Accordion expand | 200ms | ease-out | Left panel sections |
| Panel resize | real-time | none | Dragging panel dividers |
| Listen mode fade-in | 300ms | ease-out | Background darkening |
| Orb core pulse | 2000ms loop | ease-in-out | Scale 1→1.1→1 |
| Orb ambient breathe | 4000ms loop | ease-in-out | Opacity 0.1→0.2→0.1 |
| Orb active pulse | 1200ms loop | ease-in-out | Scale 1→1.3→1 (when recording) |
| Section select border | 150ms | ease-out | Dashed border appears |
| Pipeline step complete | 200ms | spring | Scale 0.9→1 + green fill |
| Pipeline spinner | 1000ms loop | linear | Continuous rotation |
| Typewriter character | 30ms per char | linear | AI response text |
| Cursor blink | 1000ms loop | step-end | Terminal cursor in log |
| Config change flash | 200ms | ease-out | Brief highlight when a value changes from LLM |

### 6.3 Reduced Motion

All animations must respect `prefers-reduced-motion`:
- Disable all loops (orb pulse, spinner, cursor blink)
- Reduce transitions to instant (0ms)
- Keep opacity changes (they don't cause motion sickness)
