**Phase:** 1.3 (Shell Polish & Presets)  
**Component:** `src/components/shell/MasterSettingsDialog.tsx`  
**Schema Impact:** `MasterConfig` root-level `settings` object (additive — no existing schema changes)  
**Documents Reference:** Doc 2 §5.1 (data model), Doc 4 §4.1 (TopBar spec), Doc 5 §5 (styling rules)

---

## 1. Feature Overview

A global **Master Settings** dialog accessible from the TopBar gear icon. This dialog controls page-level accessibility and appearance options that affect both the Reality Canvas preview and the exported JSON.

**Dual purpose:**
1. **Builder preview** — settings immediately change how the Reality tab renders sections
2. **Exported output** — settings persist in the JSON so downstream consumers (Claude Code, agentic systems) can apply them to the production site

**UX principle:** Open → Select Options → Close. All changes apply immediately to preview. No separate "save" step — closing the dialog is saving.

## 3. Schema Updates

### 3.1 Add `settings` to MasterConfig

Global settings live at the **root level** as a `settings` object — separate from the vibe/section system. Switching vibes (Warm → Ocean) does NOT reset accessibility preferences.

**File:** `src/lib/schemas/masterConfig.ts`

```typescript
import { z } from 'zod';

export const siteSettingsSchema = z.object({
  // Appearance
  appearance: z.enum(['light', 'dark']).default('light'),
  
  // Accessibility
  textScale: z.enum(['default', 'large', 'accessible']).default('default'),
  contrast: z.enum(['standard', 'enhanced', 'high']).default('standard'),
  reduceMotion: z.boolean().default(false),
  
  // Export options
  a11yWidget: z.boolean().default(false),
});

export type SiteSettings = z.infer<typeof siteSettingsSchema>;

// Updated MasterConfig
export const masterConfigSchema = z.object({
  spec: z.literal('aisp-1.2'),
  page: z.string().default('index'),
  version: z.string().default('1.0.0-RC1'),
  settings: siteSettingsSchema.default({}),  // ← NEW
  sections: z.array(sectionSchema).default([]),
});
```

### 3.2 Naming Decisions

| Original Proposal | Final Name | Reason |
|-------------------|-----------|--------|
| `mode: "light" \| "dark"` | `appearance` | Avoids collision with `uiStore.complexityMode` (DRAFT/EXPERT) and `uiStore.interactionMode` (LISTEN/BUILD). Three different `mode` properties guarantees agent confusion. |
| `textSize: "xl"` | `textScale: "accessible"` | `textSize` sounds like a pixel value. `textScale` is clearly a multiplier. "xl" is a Tailwind size class — "accessible" states the intent explicitly. |
| `contrastLevel: "minimal"` | `contrast: "high"` | **Critical fix.** The original said "minimal" means 3-color *high* contrast — but agents reading `"minimal"` will implement less contrast. Enum ascending: `"standard"` → `"enhanced"` → `"high"`. |
| *(missing)* | `reduceMotion` | **Added from WCAG 2.2/2.3.3 research.** Best practice is to provide an in-app toggle alongside respecting OS `prefers-reduced-motion`. Users may be unaware of the OS setting, may be on shared computers, or may want motion on some sites but not others. |
| `includeAccessibilityWidget` | `a11yWidget` | Shorter. Industry convention (`a11y` = accessibility). |
| Location: `theme.ts` | `masterConfig.settings` | Vibes cascade to section styles. Accessibility preferences are orthogonal — they must survive vibe switches. |

### 3.3 CSS Mapping (Exact Values)

| Setting | Value | CSS Variable / Effect | WCAG Reference |
|---------|-------|----------------------|----------------|
| `textScale: "default"` | `--hb-preview-base: 1rem` (16px) | Standard rendering | — |
| `textScale: "large"` | `--hb-preview-base: 1.125rem` (18px) | 12.5% scale increase | 1.4.4 Resize Text (AA) |
| `textScale: "accessible"` | `--hb-preview-base: 1.25rem` (20px) | 25% scale, min touch target 44px | 2.5.8 Target Size (AAA) |
| `contrast: "standard"` | No modification | Full vibe palette | — |
| `contrast: "enhanced"` | Boost saturation + contrast | All text meets 4.5:1 minimum | 1.4.3 Contrast (AA) |
| `contrast: "high"` | 3-color override: white bg, black text, vibe primary accent only | All text meets 7:1 ratio | 1.4.6 Contrast Enhanced (AAA) |
| `reduceMotion: true` | `--hb-preview-animation: none` | Disables all transitions, animations, scroll effects in preview | 2.3.3 Animation from Interactions (AAA) |
| `reduceMotion: false` | `--hb-preview-animation: normal` | Animations render normally | — |
| `appearance: "light"` | Light backgrounds per vibe | — | — |
| `appearance: "dark"` | Dark backgrounds per vibe | — | 1.4.3 applies to dark mode too |
| `a11yWidget: true` | **No visual effect in builder** | JSON flag for downstream code gen | — |

### 4.3 Styling (per Doc 4 and Doc 5)

- Dialog background: `--hb-bg` (#faf8f5)
- Dialog border: 1px solid `--hb-border`
- Dialog border-radius: 12px
- Section labels: font-mono, 12px, uppercase, tracking-wide, `--hb-text-secondary`
- Description text: DM Sans, 12px, `--hb-text-muted`
- ToggleGroup items: `--hb-surface` bg, `--hb-border` border, 8px radius
- ToggleGroup selected: `--hb-accent` border (2px), `--hb-accent-light` bg
- Switch: shadcn Switch, `--hb-accent` when on
- Section divider: 1px solid `--hb-border`, 16px vertical margin
- No custom CSS files (Doc 5 §5.2)

### 4.4 Behavior

Every change writes to `configStore.applyPatch()` immediately. No save button. Closing = done.

| Control | configStore Path | Preview Effect | Latency |
|---------|-----------------|---------------|---------|
| Appearance | `settings.appearance` | Preview switches light/dark | < 100ms |
| Text scale | `settings.textScale` | Base font size changes | < 100ms |
| Contrast | `settings.contrast` | Palette simplifies progressively | < 100ms |
| Reduce motion | `settings.reduceMotion` | All preview animations stop | < 100ms |
| A11y widget | `settings.a11yWidget` | JSON flag only, **no visual change** | < 50ms |

### 4.5 OS Preference Detection

On first load (no saved settings), detect OS preferences for intelligent defaults:

```typescript
// In configStore initialization (before any saved state loads)
const prefersReducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
const prefersDarkMode = window.matchMedia('(prefers-color-scheme: dark)').matches;

const osDefaults: Partial<SiteSettings> = {
  appearance: prefersDarkMode ? 'dark' : 'light',
  reduceMotion: prefersReducedMotion,
};
```

Once the user explicitly changes a setting, their choice overrides OS and persists. The dialog should show the user's explicit choice, not re-read OS on every open.

## 5. Accessibility of the Dialog Itself

| Requirement | Implementation | WCAG |
|-------------|----------------|------|
| **Focus trapping** | shadcn/ui Dialog (Radix) native. Do NOT re-implement. | 2.4.3 Focus Order |
| **ESC to close** | Radix handles natively. Wire `onOpenChange`. | 2.1.1 Keyboard |
| **Return focus** | Radix returns focus to trigger. Verify `document.activeElement === gearButton` after close. | 2.4.3 |
| **ARIA** | Radix applies `role="dialog"` + `aria-modal="true"`. Add `aria-labelledby` to title. | 4.1.2 Name, Role, Value |
| **Keyboard nav** | ToggleGroup: arrow keys. Switch: Space. Tab between groups. | 2.1.1 |
| **Color independence** | Selected states use border + bg shift, not color alone. | 1.4.1 Use of Color |
| **Touch targets** | All controls ≥ 24×24 CSS px (WCAG 2.2 minimum). ToggleGroup items ≥ 36px height. | 2.5.8 Target Size |
| **Self-referential motion** | If `reduceMotion` is true OR OS `prefers-reduced-motion`, dialog opens/closes instantly — no animation. | 2.3.3 |

## 7. What This Does NOT Do

| Not This | Why |
|----------|-----|
| Change builder chrome | Builder is always warm cream (ADR-009). Only Reality tab changes. |
| Render widget in Reality tab | `a11yWidget` is export metadata. No visual effect in builder. |
| Replace vibes | Vibes = section colors. Settings = global rendering. Orthogonal — vibe switch never resets a11y. |
| Add WCAG linting | Separate feature. This is user preference selection. |
| Change DRAFT/EXPERT controls | Dialog available in all modes. Doesn't change which controls show. |
| Affect Listen Mode orb | Red orb is builder chrome, not preview content. `reduceMotion` only affects section animations. |
