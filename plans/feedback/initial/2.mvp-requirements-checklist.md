## 2. MVP SCOPE DEFINITION

### PMF Equation

Per North Star Section 1.2:

```
PMF = Builder Mode + Listen Mode + Spec Documents
(all three required -- none alone is sufficient)
```

The MVP must demonstrate all three components. Remove any one and the product fails to differentiate.

### The 6 "Aha Moments" (North Star Section 3.2)

All six must be demonstrable in the capstone:

1. Click a vibe, website appears (DRAFT mode, 2 seconds)
2. Toggle to EXPERT, full property inspector appears (same data, more control)
3. Toggle to LISTEN, theatrical dark mode with red orb (wow factor)
4. Click DATA tab, see the JSON that drives everything (engineering credibility)
5. Click XAI DOCS, see AISP specs auto-generated (invisible documentation)
6. Click WORKFLOW, see the AI pipeline in action (XAI transparency)

### Level Classification

| Level | Name | MVP Required? | Rationale |
|-------|------|:---:|-----------|
| L1 | Core Builder | **YES** | Foundation: shell, hero, JSON loop, all tabs, Listen Mode visual. Without this, nothing exists. |
| L2 | Full Site Builder | **YES** | All 8 section types needed to demonstrate "60-70% marketing site coverage" claim. Section CRUD is core. |
| L3 | Specification Engine | **YES** | Spec Documents are 1/3 of the PMF equation. XAI DOCS tab must generate real specs, not placeholders. Export package required. |
| L4 | Auth & Database | NO | Persistence via localStorage is sufficient for demo. Supabase is a production concern, not a capstone differentiator. |
| L5 | LLM Functionality | NO | Chat/LLM enhances the experience but is not required for the PMF equation. Listen Mode visual (L1) + future voice (L6) satisfy the "Listen Mode" requirement visually. However, a placeholder chat input is needed. |
| L6 | Voice Mode | NO | Voice input is the full realization of Listen Mode but requires L5. The theatrical red orb visual from L1 demonstrates the concept. Full voice is aspirational for capstone. |
| L7 | Enterprise Specs | NO | Enterprise AISP precision builds on the complete feature set. L3 specs are sufficient for capstone. |

**MVP = Levels 1 + 2 + 3** (the "open-core" per the README's description: "This open-core MVP (Apache licensed)")

Note: The README states the project is "at the MVP level" for the open-core. Levels 4-7 are progressive enhancements beyond MVP.
