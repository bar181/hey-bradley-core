# Phase 1.3b: Theme Quality Reset — 10 Distinct Themes

**Supersedes:** Phase 1.3 themes (scored 20/100 — unacceptable)
**Goal:** 10 visually distinct themes where each looks like a completely different $10M website, not a CSS variable swap.

**DoD (fixed):**
1. 10 theme presets, each visually distinct (different colors, fonts, layouts, mood, ONE memorable element)
2. Live-rendered mini-preview theme cards (not identical gray blocks)
3. At least 1 video background theme
4. At least 1 full-bleed image with overlay theme
5. At least 1 text-only minimal theme
6. At least 2 split layout themes
7. 5 hero layout variants implemented (centered, split-right, split-left, overlay, minimal)
8. Copy identical across all themes
9. Playwright screenshots all 10 — no two look alike
10. UX/UI quality gate: would someone screenshot this and share it?

## The 10 Themes
1. Stripe Flow — flowing multi-color gradient
2. Notion Warm — cream bg, warm illustration feel
3. Linear Sharp — near-black, dot grid, tight type
4. Loom Friendly — bright, split, video thumbnail
5. Vercel Prism — black + rainbow prism lines
6. Nature Calm — full background nature photo overlay
7. Studio Bold — split reverse, large portfolio image
8. Video Ambient — looping video background
9. Pastel Playful — soft pastels, floating card elements
10. Neon Terminal — near-black, neon green, monospace

## Execution Order
1. research-agent FIRST (study 10 reference sites, document findings, select images)
2. theme-agent + component-agent PARALLEL (theme JSONs + hero variants)
3. integration-agent LAST (wire, test, quality gate)

## Quality Gate (NEW)
Before completing any sub-phase: UX/UI expert review of rendered output.
- Screenshot all themes side-by-side
- "Do these look like they came from the same builder?" → answer must be "No"
- Any theme that looks generic → redesign before proceeding
