# Hey Bradley — SWARM DIRECTIVE: Theme Quality Emergency Reset

**Priority:** P0 — BLOCKING  
**Issue:** The 3 themes in Phase 1.3 are visually unacceptable. They look 15 years old. The theme cards have no contrast. The heroes are generic. The colors don't work. The fonts are default. This does not meet the "wow factor" bar for a Harvard capstone.

**This directive replaces the theme specs in `12-L1-phase-1.3-1.4-themes-layout.md`.** The 10-theme list and design standards below are the new authority.

---

## 1. WHAT'S WRONG (Be Specific)

Looking at the screenshots:

**Theme 1 (Midnight Modern):** Dark background with blue accent. The hero is just centered text on a dark gradient — no visual energy, no memorable element. The gradient is flat and generic. It looks like a default Bootstrap dark template. Compare this to Stripe's hero (animated WebGL gradient with flowing complementary colors) or Linear's hero (tight typography with subtle animated mesh). Those feel like $10M products. This feels like a tutorial exercise.

**Theme 2 (Warm Sunrise):** Light theme with a stock photo on the right. The image is randomly cropped, the text is left-aligned but doesn't breathe (not enough padding), and the white background has zero personality. Compare to Notion's hero (custom illustrations, warm pastels, playful characters) or Headspace's hero (soft colors, gentle animation, whitespace as a design element). Those feel intentional. This feels like a placeholder.

**Theme 3 (Electric Gradient):** Looks almost identical to Theme 1 but with a teal accent instead of blue. The gradient text effect is barely visible. Compare to Vercel's hero (prism light-bending effect) or Arc browser's hero (vibrant teal with bold typography and animated grid). Those feel futuristic. This feels like Theme 1 with a CSS variable change.

**The theme selection cards** are the worst part. Three cards with tiny color dots and identical layout blocks — they look like radio buttons, not theme previews. A user should be able to glance at the cards and immediately feel the mood of each theme. Currently they all look the same.

## 3. THE 10 THEMES (Redesigned from Scratch)

Each theme must pass this checklist:
- [ ] **Would someone screenshot this and share it?** If not, redesign.
- [ ] **Can you tell it apart from every other theme in 1 second?** Different layout, different colors, different mood, different energy.
- [ ] **Does it look like it was made by a designer, not generated by AI?** No generic gradients, no default Bootstrap, no unstyled system fonts.
- [ ] **Does it have ONE memorable visual element?** A gradient effect, a bold font choice, an interesting image treatment, a unique color combination.

## 4. THEME CARD REDESIGN

The current theme cards (3 tiny color dots + identical layout block) are useless for previewing themes. Redesign:

```
┌──────────────────────────────────────────┐
│  ┌────────────────────────────────────┐  │
│  │                                    │  │
│  │   Actual mini-preview of the       │  │
│  │   hero section rendered at         │  │
│  │   ~200px wide showing the          │  │
│  │   real colors, fonts, and layout   │  │
│  │                                    │  │
│  └────────────────────────────────────┘  │
│  Stripe Flow                             │
│  Dark · Centered · Gradient              │  ← Mono 11px, muted tags
└──────────────────────────────────────────┘
```

Each card must be a **real styled div** showing:
- The actual background color/gradient of the theme
- A tiny headline in the theme's font and color
- A tiny CTA button in the theme's accent color
- The overall layout direction (centered vs split)

This is NOT a screenshot — it's a live-rendered micro-component using the theme's CSS variables. When a user scrolls through 10 cards, they should immediately feel the mood of each one.

## 6. SWARM EXECUTION

### 6.1 Research Phase (MANDATORY — before any code)

One agent must:
1. Visit all 10 reference sites listed in §2
2. Screenshot each hero section
3. Document: what makes each one memorable, what colors/fonts/effects they use, what layout pattern they follow
4. Save to `docs/research/theme-design-research.md`

### 6.2 Implementation

| Agent | Tasks |
|-------|-------|
| **research-agent** | Visit reference sites, document findings, select specific Unsplash/Pexels URLs per theme |
| **theme-agent** | Create 10 theme JSON files with correct colors, fonts, layout variants |
| **component-agent** | Build all 5 hero layout variants (centered, split-right, split-left, overlay, minimal). Build theme card mini-preview component. |
| **integration-agent** | Wire theme selection, verify cascade, run Playwright for all 10 themes |

### 6.3 Quality Gate

The swarm must Playwright-screenshot all 10 themes side-by-side. If any two themes look similar, or any theme looks "generic," the swarm must redesign before requesting human review.

**The test:** Show all 10 screenshots to someone who doesn't know the product. Ask: "Do these look like they came from the same builder?" The answer should be "No — these look like 10 different websites."
