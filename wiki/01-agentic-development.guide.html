Why this matters

Hey Bradley is built using a novel agentic development approach where AI agent swarms coordinate through RuFlo v3 to execute spec-driven development phases. Unlike traditional software development, every line of code is produced by coordinated AI agents following AISP-formatted specifications that reduce ambiguity to under 2%. This is an open-core version for Bradley Ross's Harvard ALM Digital Media Design capstone, with technical SME guidance from Ruv (author of RuFlo and RuVector).

💡 What It Is

An agentic development system that uses coordinated AI agents working in parallel swarms to build software. Instead of a single developer writing code, 7-13 specialized agents — coders, reviewers, testers, architects — work simultaneously under swarm coordination. Each agent reads specification documents, produces code, and validates against quality rubrics — all orchestrated by the RuFlo platform.

🔗 Think of It Like…

Think of it like a construction site where instead of one general contractor doing everything, you have a specialized crew — an electrician, plumber, framer, and painter — all working at the same time on different parts of the building, coordinated by a project manager who makes sure everyone follows the same blueprint. The blueprint is the AISP specification, and the project manager is RuFlo.

📖 How It Works

1

Specification First

Before any code is written, 6 detailed documents define every aspect of the project: vision (North Star), technical architecture (ADRs, DDD), visual design (Design Bible), agent behavioral rules (Swarm Protocol), and file structure. The AISP Crystal Atom format — a self-validating, proof-carrying protocol designed for AI-to-AI communication — ensures that each specification achieves less than 2% ambiguity, enabling agents to interpret intent with near-perfect accuracy.

2

Phase Planning

The project is divided into 7 levels (Core Builder, Full Site Builder, Specification Engine, Auth & Database, LLM Functionality, Voice Mode, and Enterprise Specs) with 21 sub-phases. Each phase has clear goals, a Definition of Done, research requirements, a development log, a quality rubric scored on a 0-4 scale, and a Playwright testing retrospective.

3

Swarm Initialization

RuFlo v3 initializes a hierarchical-mesh swarm topology. 7-13 specialized agents are spawned in parallel via Claude Code's Task tool. Each agent is assigned a specific role — coder, reviewer, tester, architect, security auditor — and receives full context from the specification documents. AgentDB provides persistent shared memory with HNSW-indexed vector search for pattern retrieval.

4

Parallel Execution

Agents work simultaneously across their domains: coders implement components following Zod-first schemas, reviewers check quality against the Swarm Protocol's cardinal sins, testers validate with Playwright, and architects verify DDD bounded context boundaries. The raft consensus model ensures a leader agent maintains authoritative state while all agents share context through AgentDB's HNSW-indexed memory (150x-12,500x faster than linear scan).

5

Phase Validation

Each phase concludes with Playwright end-to-end testing, rubric scoring on a 0-4 scale across 205 total tracked requirements spanning four quality dimensions, and a retrospective. Human review gates at phase boundaries ensure quality before proceeding. The bar for the Hey Bradley capstone: “This looks like a funded startup product, not a student project.”

✍️ The Blueprint That Builds Itself

Every piece of software begins as an idea in someone's head, and the first thing that happens to that idea is that it gets misunderstood. A product manager writes a specification, a developer reads it, and somewhere between the writing and the reading, meaning slips through the cracks. Studies suggest that natural language specifications carry 40-65% ambiguity — meaning nearly half the intent gets lost in translation. The developer fills in the gaps with assumptions, ships code that almost matches what was wanted, and the cycle of revision begins. Bradley Ross's capstone project at Harvard takes a fundamentally different path.

At the heart of Hey Bradley's development sits AISP — the AI Specification Protocol created by Bradley Ross — a formal language designed not for humans to read casually, but for AI agents to parse with mathematical precision. Where natural language leaves room for interpretation, AISP's Crystal Atom format uses Category Theory and Natural Deduction to achieve less than 2% ambiguity. When you combine this with RuFlo v3, a multi-agent swarm orchestration platform created by technical SME Ruv, something remarkable happens: instead of one developer interpreting a vague spec, 7-13 specialized AI agents read the same unambiguous blueprint and execute in parallel. Coders write components. Reviewers check quality in real time. Testers validate with Playwright. Architects enforce domain boundaries. All simultaneously, all from the same source of truth. The gap between what was intended and what gets built shrinks to nearly zero.

This is not simply faster development — it is a fundamentally different relationship between human intent and machine execution. The 21-phase roadmap with rubric scoring and retrospectives ensures that speed does not come at the cost of quality. The open-core version, licensed under Apache, demonstrates that enterprise-grade agentic development is not locked behind proprietary walls. It is accessible, reproducible, and built to be studied. For a capstone project, it sets a new standard for what is possible when specifications become precise enough to execute themselves.

📎 Additional Details

The agentic development system is built on an open-core license (Apache) and leverages three foundational tools: RuFlo v3 for multi-agent swarm coordination with hierarchical-mesh topology, RuVector for HNSW-indexed vector search enabling 150x-12,500x faster pattern retrieval than linear scan, and AgentDB for persistent shared memory across agent sessions. Technical SME Ruv (author of RuFlo and RuVector) provides architectural guidance alongside Bradley Ross's product vision.

The system tracks 205 requirements across 4 quality dimensions using a 0-4 rubric scale. Agent types span 60+ specializations including core development (coder, reviewer, tester, planner, researcher), security (security-architect, security-auditor), and swarm coordination (hierarchical-coordinator, mesh-coordinator, adaptive-coordinator). The 3-tier model routing system (ADR-026) optimizes cost and latency: Tier 1 uses WASM Agent Booster for sub-millisecond simple transforms at zero cost, Tier 2 routes to Haiku for low-complexity tasks, and Tier 3 escalates to Sonnet/Opus for complex reasoning and architecture decisions.

Requirements. This system requires Claude Code with MCP integration and the @claude-flow/cli package. The agentic approach is designed for AI-native development workflows and may not be suitable for traditional development teams without AI tooling experience.
