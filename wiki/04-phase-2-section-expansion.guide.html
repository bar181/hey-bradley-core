♿ Accessibility Fixes

Aria-labels on all interactive elements, visible focus indicators, and a minimum 12px font size enforced site-wide.

🛠 Architecture

Phase 2 established the per-section pattern that scales to any number of section types and variants.

  • JSON Theme JSONs serve as full config replacement, not partial overlays. Switching themes replaces the entire MasterConfig sections array with theme-specific content. ADR-018
  • COMP Per-section pattern: every section type has a schema definition, a renderer component, and a SIMPLE tab editor. Adding a new section type means adding three files.
  • CSS 6-slot palette maps to CSS custom properties, which Tailwind classes reference. Theme switch updates 6 vars and the entire preview re-paints. ADR-019 / ADR-021
  • HIER Section routing via SimpleTab.tsx dispatch. The SIMPLE tab reads the selected section’s type and renders the matching editor component. No routing library needed.

✓ What Works Well

The strongest outcomes from Phase 2 development.

  • 🎨 Theme switching. 10 themes produce 10 visually distinct websites. Each theme replaces layout, content, colors, and fonts — not just a CSS skin swap.
  • ✚ Section CRUD. Add any section type with real sample content from the current theme. Duplicate, reorder, or remove sections with a single click.
  • 👁 Editor wiring. Edit copy in the SIMPLE tab and see it update in the preview immediately. The JSON-to-preview pipeline works reliably across all 8 section types.
  • 📋 Scope management. The human-1.md correction expanded scope early, avoiding a Phase 3 bottleneck. Catching architectural gaps in Phase 2 prevented compounding delays later.

⚠ Known Gaps

Issues identified during Phase 2 that remain unresolved.

One variant per non-hero section. Each non-hero section type has a single renderer variant. No FeaturesCards, CTASplit, or other layout alternatives exist yet.
Font cascade incomplete. The font family selection does not reliably reach all section renderers. Some sections fall back to the browser default instead of the theme font.
Light mode preview contrast. Light mode palette produces poor text contrast in several section types, making content difficult to read.
No full-page preview mode. The preview shows one section at a time. There is no way to see the entire page layout as a user would experience it.
Dead colors blocks in theme JSONs. Legacy color definitions remain in theme files alongside the 6-slot palette, causing confusion about which values are actually used.
