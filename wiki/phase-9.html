Phase Overview

Phase 9 bridges Stage 1 (Presentation) and Stage 2 (Pre-LLM MVP) by shipping builder features that work without any AI backend.

Mission: Make the builder usable by a non-technical person -- your grandmother should be able to pick a template, upload her logo, customize colors, save her project, and get professional specifications without ever touching code.

Grandma UX

Template-first onboarding, jargon cleanup across 14 files, accordion defaults collapsed, auto-select hero section after example load. The builder now speaks human.

Five New Features

Image upload (drag-drop, base64, 2MB limit), project save/load (localStorage + JSON export), custom hex colors with native picker, SEO fields, brand management (logo, favicon, OG image).

Type Safety Overhaul

Eliminated all 144 as any casts with a new sectionContent.ts helper library. TypeScript now catches real bugs across 38 template files.

Professional Spec Rendering

Replaced raw <pre> tags with react-markdown + remark-gfm. Specs render with real tables, headings, bold text, and code blocks. Documented in ADR-030.

Architecture Decisions

Key technical decisions made during Phase 9.

ADR-030: Spec Rendering Architecture

Replaced <pre> + HumanHighlighted with react-markdown + remark-gfm for all 5 markdown-based spec generators. AISP keeps its Crystal Atom renderer.

AspectBeforeAfter
Rendering<pre> monospace dumpProse with headings, tables, bold, code
StylingNone (raw text)hb-spec-prose CSS with light/dark
DependenciesNonereact-markdown, remark-gfm, @tailwindcss/typography

Type Safety: sectionContent.ts

Centralized helper library with getStr(), getItems(), and getContent() functions. Safely extracts typed data from section content objects. Eliminated all 144 as any casts across 38 files without changing runtime behavior.

Onboarding: Preview-First Design

Two-panel layout with real preview screenshots generated via Playwright (scripts/generate-previews.mjs) using fresh browser contexts. Image fallbacks hide broken images gracefully. Zustand stores require fresh contexts to avoid stale state.
