Phase 3: Full Site Builder

Onboarding page, full-page preview, navbar, font cascade fix, and the complete user flow.

Key Takeaway

Phase 3 transformed Hey Bradley from a section editor into a website builder. Users now land on an onboarding page, pick a theme, and see a complete stacked website with navigation. The SIMPLE tab was reduced from 25 controls to 14. Font cascade finally fixed. This is the biggest single-phase quality jump in the project.

⚡ What Was Built

Six major capabilities shipped across Phase 3, turning the builder from a section editor into a full website builder.

👁 Full-Page Preview

Preview/Edit toggle hides both panels, stacks all sections into a single scrollable page, and Escape exits back to the editor.

☰ Navbar Section

NavbarSimple renderer auto-links from enabled sections with a CTA button. Navigation appears at the top of the full-page preview.

🏠 Onboarding Page

The / route displays 10 theme cards in a clean grid. Click a theme to applyVibe and navigate to /builder. “Continue editing” resumes the last session.

✍ Font Cascade Fix

All 11 renderers now use var(--theme-font). The font family selection was broken since Phase 1 and finally reaches every section.

▦ Second Variants

FeaturesCards, CTASplit, and FAQTwoCol added as alternate layouts. Variant selectors wired into the SIMPLE editors for each section type.

✎ SIMPLE Tab Simplification

Hero editor reduced from ~25 controls to ~14. Light/Dark/Auto toggle moved into the Theme section for a cleaner editing experience.

🛠 The User Flow

Phase 3 established the complete end-to-end user journey from landing to full-page preview.

  • 1 Land on / — see 10 theme cards in a clean grid. Each card shows a preview of the theme’s colors and typography.
  • 2 Click a theme — applyVibe loads the theme JSON, then navigate to /builder with the selected theme active.
  • 3 3-panel editor — left panel (sections list), center (live preview), right panel (SIMPLE/Expert editor) for the selected section.
  • 4 Click Preview — both panels hide, all sections stack into a full scrollable website with the navbar at the top.
  • 5 Press Escape — return to the 3-panel editor to continue editing any section.

⚠ Known Gaps

Issues identified during Phase 3 that remain unresolved.

No smooth scroll in preview navbar links. Clicking a navbar link in full-page preview jumps to the section without a smooth scroll animation.
Only 1-2 variants per non-hero section. Most section types still have limited layout options. More variants are needed for real design diversity.
Navbar not in all 10 theme JSONs. Only SaaS and default themes include the navbar section definition. Other themes lack navbar configuration.
No drag-and-drop reorder. Sections can only be reordered via up/down buttons. Visual drag handles are not yet implemented.
“Auto” mode is cosmetic only. The Light/Dark/Auto toggle includes an Auto option, but it does not detect the system color scheme preference.
