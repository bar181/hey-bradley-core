Why this matters

Hey Bradley is a JSON-driven marketing website specification platform — NOT a website builder. It produces two outputs simultaneously: a live visual preview the user interacts with, and enterprise-grade AISP specification documents the user exports. The pipeline is: Ideation → Hey Bradley → Specs + JSON → Claude Code → Production Site. This open-core MVP (Apache licensed) is Bradley Ross's Harvard ALM Digital Media Design capstone, designed to demonstrate that specification-driven development with AI agents can produce enterprise-quality output.

💡 What It Is

A smart whiteboard that listens to what you want, shows you a website preview in real-time, and secretly writes detailed technical specifications behind the scenes. You describe your marketing website — pick a style, choose sections, type or speak your content — and Hey Bradley produces both a beautiful preview AND a specification package that AI tools like Claude Code can turn into a production website. It bridges the gap between “I want a website” and “here are the exact specs to build it.”

🔗 Think of It Like…

Think of it like an architect who sketches your dream house in real-time as you describe it, while simultaneously writing the complete construction blueprints in the background. When you’re done talking, you walk away with both a 3D model to show your family AND a blueprint package that any builder can use to construct the exact house you described. Hey Bradley is that architect — except for marketing websites, and the “builder” is an AI agent like Claude Code.

  • Non-technical users can’t participate in spec creation

Our Approach

  • AISP specs with < 2% ambiguity generated automatically
  • Single platform: preview + data + specs + workflow
  • “Grandson shows grandma” — anyone can use Draft mode
Trade-off acknowledged. Hey Bradley covers ~60–70% of marketing websites (landing pages, agency sites, SaaS pages) but intentionally excludes e-commerce, blogs, dashboards, and web apps. It is a specification platform, not a full website builder — it feeds builders like Claude Code rather than replacing them. If you need a Shopify store or a blog CMS, Hey Bradley is not the right tool.

📖 How It Works

1

Pick a Vibe

Choose from preset themes (Warm, Ocean, Forest) that cascade across 15+ JSON nodes with a single click. The entire site preview updates instantly. Even non-technical users (“grandma”) can do this in Draft mode. Each vibe defines colors, typography, and section style overrides — one click transforms the entire look and feel.

2

Build with Sections

Add, remove, and reorder 8 section types: Hero, Features, Pricing, CTA, Footer, Testimonials, FAQ, and Value Props. Each section type has 3–5 visual variants. In Draft mode, you pick from visual cards. In Expert mode, you get a full CSS property inspector with direct control over layout, content, and style values.

3

See Everything at Once

Four center tabs show different views of the same data: REALITY (live preview), DATA (raw JSON with syntax highlighting), XAI DOCS (auto-generated AISP specs), and WORKFLOW (AI pipeline visualization). Toggle between DRAFT (simple) and EXPERT (full control) at any time — the data never changes, only the controls do.

4

Talk to It

In Listen Mode, a theatrical dark overlay with a pulsing red orb appears. Speak naturally and the AI translates your words into website changes every ~15 seconds. The site builds itself as you talk — this is the “aha moment” that demonstrates voice-first specification creation is not just possible but intuitive.

5

Export Specs

Download a complete specification package: AISP-formatted specs, JSON config, and human-readable documentation. Paste the output into Claude Code to generate a production-ready website that matches your preview exactly. The JSON is the single source of truth — what you see IS what gets specified.

✍ The Whiteboard That Writes Its Own Blueprint

You know exactly what you want your website to feel like. You can see it in your head — the colors, the energy, the way it should make a visitor feel when they land on it. But the moment you try to explain that vision to a designer, something breaks. “Clean, modern, maybe some orange accents” enters a translation pipeline where a designer interprets the brief into wireframes (making dozens of assumptions), a developer interprets the wireframes into code (making dozens more), and by the time the site launches, your vision has been translated three times. You look at the result and say, “That is not what I meant.” You are not bad at describing what you want. The system is bad at preserving what you mean.

That gap — between the website in your head and the website on the screen — is not just a practical problem. It makes you feel powerless. You start second-guessing your own taste. You settle for “close enough.” You tell yourself maybe you are asking for too much. You are not. The tools just were not built for you. Hey Bradley exists as the guide in your story: you speak or click, and the system simultaneously shows you what it understood (the live preview) and writes the technical specification (the AISP documents) behind the scenes. No designer interpretation step. No developer interpretation step. What you see IS what gets specified.

When a grandmother can click three buttons and unknowingly produce an enterprise-grade AISP specification that Claude Code turns into a production website, the hero wins. Not because the technology is clever — but because the hardest technical work disappears behind the simplest possible interface. The person with the vision stays in control from start to finish, and the result is not a compromise. It is exactly what they imagined.

📎 Additional Details

Tech stack: Vite + React 18 + TypeScript + Tailwind CSS + shadcn/ui + Zustand + Zod. The architecture follows Domain-Driven Design with four bounded contexts: Builder (UI rendering), Intelligence (LLM and voice), Specification (doc generation), and Persistence (storage).

Design system: “Warm Precision” — cream backgrounds (#faf8f5), orange accent (#e8772e), monospace labels (JetBrains Mono), warm dark brown text (#2d1f12). Listen mode switches to a theatrical dark overlay (#0a0a0f) with a pulsing red orb (#ef4444).

Four user personas: Grandson (non-technical, Draft + Build), Framer User (power user, Expert + Build), Dad (needs specs for Claude Code, Expert + Build), and Enterprise (needs AISP precision, Expert + Listen).

Subject matter expert advisors: Reuven Cohen and Bence Csernak.

License: Open-core under Apache license — MVP level.

Important distinction. Hey Bradley is a specification platform, not a website builder. It does not host websites, manage domains, or provide CMS functionality. It produces specifications that feed into production tools like Claude Code. If you need hosting, domain management, or content management, you will need a separate service downstream.
